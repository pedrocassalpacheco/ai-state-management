# AI Chatbot Simulation System

Simulates realistic conversations between users and AI chatbots with full database persistence, conversation memory, and vector embeddings.

## Architecture

```
┌─────────────┐
│   Users     │
│  (5 users)  │
└──────┬──────┘
       │
       ├──────────────┐
       │              │
       ▼              ▼
  ┌─────────┐   ┌─────────┐
  │ ChatBot │   │ ChatBot │  ... 5 different bots
  └────┬────┘   └────┬────┘
       │              │
       ├──────────────┤
       ▼              ▼
┌──────────────────────────┐
│   Session Manager        │  Create/update sessions
│   Message Handler        │  Store messages + embeddings
│   Conversation Memory    │  Create snapshots
└───────────┬──────────────┘
            │
            ▼
    ┌───────────────┐
    │  TiDB (MySQL) │
    │  - sessions   │
    │  - messages   │
    │  - snapshots  │
    └───────────────┘
```

## Components

### 1. **ChatBot** (`bot.py`)
- Manages individual bot personality and behavior
- Maintains sliding window conversation memory (last 6 messages)
- Uses DeepSeek model to generate contextual responses
- Stays in character based on bot type and system prompt

### 2. **SessionManager** (`session_manager.py`)
- Creates new conversation sessions in database
- Tracks metrics: message count, token usage, timestamps
- Archives sessions when conversation ends

### 3. **MessageHandler** (`message_handler.py`)
- Inserts messages into database
- Generates vector embeddings using nomic-embed-text
- Supports both normalized and denormalized schemas

### 4. **ConversationMemory** (`memory.py`)
- Creates memory snapshots every 5 messages
- Generates summaries and extracts key facts
- Stores embeddings for semantic search

### 5. **ConversationSimulator** (`simulator.py`)
- Orchestrates multi-user conversations
- Generates realistic questions and follow-ups
- Manages conversation flow and timing

## Bot Types

- Helps with software/hardware troubleshooting
- Provides step-by-step solutions

### 2. **Learning Tutor** (`learning-tutor`)

### 4. **Career Coach** (`career-coach`)
- Professional development mentor

### 5. **Creative Writer** (`creative-writer`)
- Writing assistance and storytelling
- Character and plot development

## Features

✅ **Real-time Persistence**: All messages saved to database immediately  
✅ **Conversation Memory**: Bots remember recent context (sliding window)  
✅ **Vector Embeddings**: Every message embedded with nomic-embed-text  
✅ **Memory Snapshots**: Periodic summaries for long-term memory  
✅ **Schema Flexibility**: Works with both normalized and denormalized schemas  
✅ **Natural Conversations**: AI-generated questions and responses  
✅ **Session Management**: Full lifecycle tracking

## Usage

### Basic Simulation (5 users)

```bash
# Use default database (ai_memory)
python3 -m chatbot.simulator

# Or run directly
python3 chatbot/simulator.py
```

### Specify Database

```bash
# Use normalized schema (OLAP)
python3 -m chatbot.simulator ai_memory

# Use denormalized schema (OLTP)
python3 -m chatbot.simulator ai_memory_colocated

# Or with environment variable
TIDB_DATABASE=ai_memory_colocated python3 -m chatbot.simulator
```

### Custom Number of Users

```bash
# Simulate with 10 users
python3 -m chatbot.simulator ai_memory 10
```

## Conversation Flow

```
1. Session Created
   └─> New session in database with user_id + bot_id

2. User asks question (Turn 1)
   ├─> Question generated by AI based on bot type
   ├─> Saved to messages table
   └─> Embedding generated and stored

3. Bot responds
   ├─> Response generated using DeepSeek + conversation history
   ├─> Saved to messages table
   ├─> Embedding generated and stored
   └─> Session updated (message_count++, tokens++)

4. Repeat for N turns (default: 10)
   └─> Follow-up questions based on conversation context

5. Memory Snapshot (every 5 messages)
   ├─> Summary generated
   ├─> Key facts extracted
   ├─> Embedding created
   └─> Saved to memory_snapshots table

6. Session Archived
   └─> Status changed to 'archived', ended_at set
```

## Database Schema Support

### Normalized Schema (`ai_memory`)
```sql
messages (
    message_id,
    session_id,  -- user_id/bot_id via sessions
    role,
    content,
    ...
)
```

### Denormalized Schema (`ai_memory_colocated`)
```sql
messages (
    message_id,
    session_id,
    user_id,     -- Denormalized for partitioning
    bot_id,      -- Denormalized for partitioning
    role,
    content,
    ...
) PARTITION BY KEY(user_id, bot_id);
```

The simulator **automatically detects** which schema is in use and adapts!

## Configuration

Edit `chatbot/config.py` to customize:

- `NUM_CONVERSATION_TURNS`: Messages per conversation (default: 10)
- `MEMORY_SNAPSHOT_INTERVAL`: Snapshot frequency (default: 5 messages)
- `MEMORY_WINDOW_SIZE`: Context window size (default: 6 messages)
- `CHAT_MODEL`: Ollama model for chat (default: deepseek-r1:1.5b)
- `EMBEDDING_MODEL`: Embedding model (default: nomic-embed-text)

## Verifying Results

### Check Sessions
```sql
SELECT session_id, user_id, bot_id, message_count, total_tokens, status
FROM sessions
ORDER BY started_at DESC
LIMIT 10;
```

### Check Messages
```sql
SELECT message_id, role, LEFT(content, 50) as content_preview, tokens_used
FROM messages
WHERE session_id = '<session_id>'
ORDER BY created_at;
```

### Check Memory Snapshots
```sql
SELECT snapshot_id, LEFT(summary, 100) as summary_preview, 
       message_count, importance_score
FROM memory_snapshots
ORDER BY created_at DESC
LIMIT 10;
```

## Requirements

- Python 3.8+
- TiDB cluster running (or MySQL)
- Ollama with models:
  - `deepseek-r1:1.5b` (chat)
  - `nomic-embed-text` (embeddings)
- Python packages:
  - `pymysql`
  - `ollama`

```bash
# Install dependencies
uv add pymysql ollama

# Pull Ollama models
ollama pull deepseek-r1:1.5b
ollama pull nomic-embed-text
```
